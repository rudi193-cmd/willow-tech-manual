## Why this edition exists

This tab is shaped like a **Honda powersports Owner’s Manual**: **Vehicle safety** first, then **how to operate**, then **owner maintenance**, then **if something goes wrong**, then **warranty-style information**, then **hard numbers**. The inspiration is the real document family Honda ships (for example modern manuals use a **Vehicle safety → Operation guide → Maintenance → Troubleshooting → Information → Specifications** spine).

It is named **Elite** after Sean’s first scooter — a **1986 Honda CH-80** — and the moment **right after the purchase**: opening the glove box and finding the **original owner’s book** and **tool kit** still there. That is the user experience we are stealing: *orientation and agency included*, not a workshop teardown.

## Not a clone

We do **not** reproduce Honda copyrighted text or diagrams. We only mirror the **section pattern** so muscle memory from any Honda owner’s booklet transfers.

## Canon

- **Production stack:** `willow-1.9`
- **Forward / R&D:** `willow-sandbox` (2.0 line)
- **Deep repair / strip / rebuild:** [Workshop (Sticky spine)](/workshop) tab

## Read order (Honda pattern)

1. [Vehicle safety](/owners-manual/vehicle-safety) — read before you touch hot paths.
2. [Operation guide](/owners-manual/operation-guide) — normal daily use of the stack.
3. [Maintenance](/owners-manual/maintenance) — what you are expected to do on a schedule.
4. [Troubleshooting](/owners-manual/troubleshooting) — short symptom trees.
5. [Information](/owners-manual/information) — warranties, manuals, reporting.
6. [Specifications](/owners-manual/specifications) — capacities, versions, IDs.
7. [Quick reference](/owners-manual/quick-reference) — one screen of “the usual numbers.”

## Permanent part of the vehicle (metaphor)

Treat this edition like Honda treats the paper book: **keep it with the deployment** — linked from the repo README, the runbook binder, and the onboarding path for new operators.
